## Current Review Diagnosis

The three reviews converge on four blocking issues:

1. **Teacher-coupled circularity:** Current BC demonstrations are generated by a variable-impedance teacher, so the manuscript cannot cleanly claim variable impedance has independent physical value without matched fixed-teacher or motion-matched controls.
2. **3DoF sim-only external validity:** Current claims are safe only as a controlled benchmark/diagnostic study; claims about industrial insertion, real-world guidance, or general VI learning must be removed unless a new orientation/cross-sim stress layer is added.
3. **Jargon and contribution inflation:** SG-VI and SCI are presented too prominently for benchmark-local constructs. Internal process terms such as Sprint 4A must disappear from paper-facing prose.
4. **Mechanics analysis gap:** Figure 3 and the high-friction interpretation need success/failure separation, contact-work/force-position summaries, and a clearer causal boundary between impedance scheduling and PPO instability.

## Revised Paper Spine

Do not write the revision as a checklist response. The new manuscript spine is:

```text
We study contact-rich insertion not as an algorithm leaderboard, but as a support-gated learnability problem. Demonstration support controls contact entry; SCI audits whether rollouts remain in demonstration-supported contact-relevant regions; and variable impedance contributes primarily through lower-load contact paths once support, motion prior, and impedance prior are separated.
```

This spine implies three paper-level claims:

1. **Learnability gate:** In this reduced 3DoF contact-insertion benchmark, demonstration support is the dominant factor that enables policies to enter recoverable contact.
2. **SCI diagnostic:** SCI is a benchmark-local support-overlap diagnostic whose usefulness must be supported by bin sensitivity, alternative-metric audits, and correlation with contact/success/failure metrics.
3. **Variable-impedance physical role:** Variable impedance is not claimed to be universally superior in success rate. Tuned fixed impedance can succeed; VI's defensible value is lower-load/lower-work contact behavior under high-friction, motion-matched, or success-matched conditions.

```python
from __future__ import annotations

from dataclasses import dataclass

@dataclass(frozen=True, slots=True)
class TeacherCouplingCondition:
    condition_name: str
    demo_teacher_preset: str
    bc_demo_policy_name: str
    policy_family: str
    total_timesteps: int
    seed: int


def build_teacher_coupling_grid(*, seeds: list[int], total_timesteps: int) -> list[TeacherCouplingCondition]:
    specs = [
        ("vv_demo_vi_policy", "teacher_variable_variable", "variable_impedance", "variable_impedance"),
        ("vf_demo_fi_policy", "teacher_variable_fixed", "fixed_impedance", "fixed_impedance"),
        ("vv_demo_fi_policy", "teacher_variable_variable", "fixed_impedance", "fixed_impedance"),
        ("pf_demo_fi_policy", "teacher_pose_fixed", "fixed_impedance", "fixed_impedance"),
        ("pv_demo_vi_policy", "teacher_pose_variable", "variable_impedance", "variable_impedance"),
    ]
    return [
        TeacherCouplingCondition(name, preset, demo_policy, family, total_timesteps, seed)
        for seed in seeds
        for name, preset, demo_policy, family in specs
    ]
```
